# 00 Rules Summary

## Детерминизм
- Фиксированный порядок колонок/строк, вычисление `hash_business_key` и `hash_row` по схемам.
- Атомарная запись файлов (temp → replace), контрольные суммы в `checksums/`.

## Валидация перед записью
- Каждая таблица проходит Pandera-валидацию; несоответствия типов или колонок считаются ошибкой сборки.
- Политика validate-before-write обязательна для всех пайплайнов.

## Базовые компоненты
- Используйте `UnifiedLogger` для структурированного логирования и прогресса.
- Доступ к API через `UnifiedAPIClient` с ретраями, лимитами и кэшем.

## Именование и тестирование
- Соблюдайте правила именования модулей/классов/файлов конфигурации (см. styleguide).
- Минимальные требования к тестам: покрытие ≥85% ключевых модулей, юнит/интеграционные/golden-тесты для критичных трансформаций.

## Ограничения количества классов
- **Zero-sum class count**: ЛЮБОЕ ДОБАВЛЕНИЕ КЛАССА (кроме ABC) должно сопровождаться исключением другого класса.
- Общее кол-во классов (кроме ABC) не может увеличиваться без прямого приказа пользователя.
