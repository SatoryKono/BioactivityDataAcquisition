# 00 Schemas Overview

## Pandera-схемы
- **ActivitySchema** — бизнес-ключ активности, числовые поля значений, идентификаторы Assay/Target/TestItem, контроль типов единиц.
- **AssaySchema** — идентификаторы ассеев, тип эксперимента, организм, ссылки на публикации.
- **TargetSchema** — идентификаторы таргетов (ChEMBL, UniProt), типы мишеней и классификации.
- **TestitemSchema** — идентификаторы соединений (ChEMBL, PubChem), структуры (SMILES/InChI), связи родитель/потомок.
- **DocumentSchema** — метаданные публикаций (DOI/PMID), ссылки на источники, статусы обогащения.

## Использование
Схемы применяются в стадии validate через ValidationService. `column_order` задаёт фиксированный порядок для записи и вычисления хешей. Бизнес-ограничения (уникальность ключей, диапазоны значений, обязательность идентификаторов) обеспечивают стабильность и сопоставимость данных.
