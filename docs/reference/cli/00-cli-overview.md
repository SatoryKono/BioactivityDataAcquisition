# CLI Architecture

Интерфейс командной строки (CLI) построен на библиотеке **Typer**. Он обеспечивает унифицированную точку входа для запуска пайплайнов, валидации конфигов и сервисных задач.

## Компоненты

### CLICommandABC
**Модуль:** `src/bioetl/cli/base.py`

Абстрактный базовый класс для всех команд. Требует реализации метода `register(app)`, который добавляет команду в Typer-приложение.

### Entry Point
**Файл:** `src/bioetl/cli/main.py`

Точка входа приложения. Собирает все зарегистрированные команды и запускает `typer.run()`.

## Принципы

1. **Explicit Arguments**: Все обязательные параметры передаются как аргументы, опциональные — как флаги (Options).
2. **Config Override**: Поддержка флага `--set` для точечного переопределения параметров конфигурации.
3. **Structured Output**: Использование Rich для красивого вывода в консоль, но логи пишутся через UnifiedLogger.
