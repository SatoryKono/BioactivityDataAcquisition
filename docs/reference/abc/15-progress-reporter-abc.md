# 15 Progress Reporter ABC

## Описание

ProgressReporterABC описывает интерфейс для агрегированной отчётности о ходе выполнения пайплайна. Он предоставляет методы для фиксации количества извлечённых, валидных и отброшенных записей, а также причин их отбрасывания. Реализация может агрегировать эти данные в памяти, отправлять их в внешнюю систему мониторинга или записывать в лог, но контракт не навязывает конкретный механизм. ProgressReporterABC не должен принимать решений о судьбе данных и не должен дублировать логику валидаторов или DQ-правил; он только фиксирует факты, которые ему передаёт оркестратор или стадии. Наличие этого интерфейса позволяет отделить сбор статистики от бизнес-логики и получить единый источник правды о том, что реально происходило во время запуска.

## Interface

```python
from typing import Protocol

class ProgressReporterABC(Protocol):
    def record_extracted(self, count: int) -> None:
        """Зафиксировать количество извлечённых записей."""
        ...

    def record_valid(self, count: int) -> None:
        """Зафиксировать количество валидных записей."""
        ...

    def record_rejected(self, count: int, reason: str) -> None:
        """Зафиксировать количество отброшенных записей с причиной."""
        ...
```

## Related Components

- **PipelineHookABC**: может использовать reporter для фиксации событий
- **DQRuleABC**: может использовать reporter для фиксации проблем качества
- **ValidatorABC**: может использовать reporter для фиксации ошибок валидации
