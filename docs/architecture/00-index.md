# Architecture Index

## Обзор
Архитектура **BioETL** спроектирована для обеспечения надежности, расширяемости и детерминизма при работе с научными данными. Она базируется на слоистом подходе, строгой типизации и ABC-контрактах.

## Структура раздела

1. **[Domain Objects](01-domain-objects.md)**
   Описание ключевых сущностей системы: `Assay`, `Activity`, `Target`, `TestItem`. Их поля, связи и бизнес-логика.

2. **[ETL Layers](02-etl-layers.md)**
   Подробное описание 6 уровней архитектуры: от оркестрации и мониторинга до трансформации и записи.

3. **[Data Flow](03-data-flow.md)**
   Жизненный цикл данных внутри пайплайна: `extract` → `transform` → `validate` → `write`. Описание хуков и режима `dry_run`.

4. **[Duplication Reduction](04-duplication-reduction.md)**
   Принципы и паттерны, используемые для снижения дублирования кода: трёхслойный паттерн ABC/Default/Impl, иерархия классов пайплайнов.

## Ключевые компоненты

- **`PipelineBase`** — основа всех ETL-процессов.
- **`UnifiedAPIClient`** — единая точка доступа к внешним API.
- **`UnifiedLogger`** — стандартизированное логирование.
- **`SchemaRegistry`** — централизованное управление схемами данных.
