# Glossary

## Доменные термины

| Термин | Описание |
|--------|----------|
| **Assay** | Описание биологического эксперимента (binding, functional, ADMET и др.). Содержит условия проведения теста. |
| **Activity** | Результат измерения биоактивности вещества в рамках конкретного ассая (например, IC50, Ki). |
| **Target** | Биологическая мишень (белок, рецептор, фермент), на которую направлено действие вещества. |
| **TestItem** | Тестируемый объект — химическое соединение, молекула, антитело. |
| **Document** | Научная публикация или патент, из которого извлечены данные об активности. |

## Архитектурные термины

| Термин | Описание |
|--------|----------|
| **PipelineBase** | Абстрактный базовый класс для всех табличных ETL-пайплайнов. |
| **Stage** | Отдельный этап обработки данных: `extract`, `transform`, `validate`, `write`. |
| **ABC** | Abstract Base Class — абстрактный базовый класс, определяющий контракт интерфейса. |
| **RunResult** | Объект, содержащий результаты выполнения пайплайна (статус, метрики, пути к файлам). |
| **StageDescriptor** | Объект, описывающий конфигурацию и порядок выполнения стадий для runtime-оркестратора. |
| **Business Key** | Набор полей, уникально идентифицирующий сущность с точки зрения бизнеса (для дедупликации). |
| **Hash Row** | Детерминированный хеш (SHA256) всего содержимого строки для отслеживания изменений. |
