# Overview

## Назначение проекта
Проект **BioETL** реализует расширяемую ETL-архитектуру для извлечения, нормализации и интеграции биологической активности и связанных научных данных из REST API научных баз.

Основная цель — создание единого нормализованного хранилища данных для построения витрин, аналитики и графов знаний.

## Поддерживаемые источники
В настоящее время полностью поддерживается:
- **ChEMBL** (Activity, Assay, Target, TestItem, Document)

Планируется интеграция:
- PubMed (metadata)
- PubChem (compounds)
- UniProt (targets)
- IUPHAR Guide to Pharmacology (interactions)
- OpenAlex (publications)
- CrossRef (DOI resolution)
- Semantic Scholar (abstracts)

## ABC-модель доменных сущностей
Все пайплайны оперируют едиными абстракциями:
- **Assay** — описание биологического эксперимента (условия, тип, организм).
- **Activity** — результат измерения биоактивности вещества в конкретном ассае.
- **Target** — биологическая мишень (обычно белок), на которую направлено действие.
- **TestItem** — тестируемый объект (молекула, соединение, антитело).
