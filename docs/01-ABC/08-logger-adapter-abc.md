# 08 Logger Adapter ABC

## Описание

LoggerAdapterABC определяет интерфейс адаптера логирования. Обеспечивает унифицированный способ структурированного логирования событий пайплайна, не зависящий от конкретной библиотеки логирования. Позволяет привязывать контекст (например, run_id, имя пайплайна) и логировать события пайплайна консистентно. Он предоставляет методы для логирования сообщений с дополнительными полями (structured logging), что позволяет агрегировать и анализировать логи в системах мониторинга. Метод `bind()` создает новый логгер/адаптер с привязанным контекстом, который будет использоваться для всех последующих сообщений (например, run_id для всех сообщений). Интерфейс не навязывает конкретную реализацию логирования, но гарантирует поддержку структурированных полей и уровней логирования.

## Interface

```python
from abc import ABC, abstractmethod
from typing import Any

class LoggerAdapterABC(ABC):
    """Интерфейс структурированного логгера."""
    
    def bind(self, **fields: Any) -> "LoggerAdapterABC":
        """Создать новый логгер с привязанным контекстом. Возвращает новый экземпляр логгера с добавленными полями контекста."""
        raise NotImplementedError
    
    @abstractmethod
    def info(self, message: str, **kwargs: Any) -> None:
        """Логировать информационное сообщение с дополнительными полями."""
        pass

    @abstractmethod
    def warning(self, message: str, **kwargs: Any) -> None:
        """Логировать предупреждение с дополнительными полями."""
        pass

    @abstractmethod
    def error(self, message: str, **kwargs: Any) -> None:
        """Логировать ошибку с дополнительными полями."""
        pass
```

## Related Components

- **PipelineHookABC**: использует логгер для уведомлений о событиях
- **ErrorPolicyABC**: использует логгер для записи ошибок
