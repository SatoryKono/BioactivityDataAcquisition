# 13 Path Strategy ABC

## Описание

PathStrategyABC инкапсулирует правила построения путей вывода данных. Он принимает абстрактный partition-контекст в виде Mapping[str, str] и возвращает Path, по которому должен быть размещён результат работы пайплайна или отдельной партии. Реализация кодирует шаблоны каталогов, имена файлов, схему партиционирования по датам, источникам, версиям и другим признакам. PathStrategyABC не выполняет запись и не управляет содержимым артефактов; он только стандартизует структуру и обеспечивает стабильность путей при повторных запусках. Выделение стратегии путей в отдельный интерфейс позволяет согласованно менять layout на диске или в объектном хранилище без переписывания writer'ов и остальной части пайплайна, а также использовать разные схемы разбиения для разных пайплайнов в единой кодовой базе.

## Interface

```python
from pathlib import Path
from typing import Mapping

class PathStrategyABC:
    def build_path(self, partition_context: Mapping[str, str]) -> Path:
        """Построить путь на основе контекста партиционирования."""
        raise NotImplementedError
```

## Related Components

- **WriterABC**: использует стратегию для определения пути записи
- **MetadataWriterABC**: может использовать стратегию для метаданных
